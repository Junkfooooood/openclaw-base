### 我验证到的“还没有真正落地”的部分

#### 1. 整条复杂任务 workflow 还不能一键跑通

最关键的问题有两个：

第一，`shared/workflows/active/dispatch.lobster.yml` 里用的是：

- `openclaw.invoke --tool llm-task ...`

但你当前这台机器上的 OpenClaw CLI **根本没有 `openclaw invoke` 这个命令**。

这意味着：

- workflow 文件写了
- Task Tree schema 写了
- dispatch 路径写了
- 但“从 workflow 自动起飞”这一步实际上是断的

第二，`dispatch -> validate -> approve` 依赖标准 JSON 管道，
但当前 CLI 在输出 JSON 之前会先打印：

- `[plugins] feishu_doc: Registered ...`
- `[plugins] feishu_chat: Registered ...`

结果就是：

- `internal-dispatch` 的 stdout 被污染
- `validator-run` 读 stdin 时会直接 JSON.parse 失败
- `approval request` 同理也会失败

也就是说：

- 这些模块单独运行是好的
- 但串起来跑时，当前输出格式不干净，整条闭环不稳定

#### 2. DAG 异步调度还没有真正实现

你要求的是：

- A 等待时，C 可以并行调度
- 存在依赖关系的 branch 按 DAG 跑

但当前 `internal-dispatch run` 的真实行为只是：

- 校验 Task Tree 结构
- 把 Task Tree 写入 `shared/runtime/queue/`
- 输出一个最小 dispatch summary
- 给每个 branch 标记 `queued`

它 **没有**：

- 真正调用 `learning / curator / executor / validator`
- 没有依赖解析执行器
- 没有并行调度器
- 没有 branch completion watcher
- 没有“等 A 完成同时先跑 C”的实际机制

所以这部分现在是：

- `DAG schema / 依赖表达能力已存在`
- `DAG runtime scheduler 尚未落地`

#### 3. “所有 agent 后台活动统一留痕”目前只完成了一半

你要求的是：

- 所有 agent 的后台活动公开透明
- 并以相同格式留痕

我检查后发现：

- live 配置里确实声明了 5 个 agent
- 但运行态的 `agents/` 目录目前只有 `agents/main`
- `learning / curator / executor / validator` 还没有真正自己的 runtime/session store

这意味着：

- 角色定义是有的
- 统一留痕格式的思路也是有的
- 但真实运行时，目前还没有形成“每个 agent 都有自己的会话和轨迹记录”的状态

所以这条要求现在只能算：

- **结构上支持**
- **运行上未完全实现**

#### 4. 黑板卡格式目前有一个真实瑕疵

我在做 `board-sync update` 之后，归档卡中发现 `Branch Status` 区域出现了重复内容：

- 一部分是新写入的 branch 状态
- 一部分是初始化时遗留的旧 branch 行

这说明当前 `board-sync update` 的 section replace 逻辑还不够稳，
你的“统一透明留痕”虽然已经有格式，但格式清洁度还不够。
