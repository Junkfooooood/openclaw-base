## 四、建议的 Feishu 开场方式

如果你的目标是让 agent 最快进入状态，最省力的开场方式是直接让它“先自报家门，再说明系统结构与当前能力”。

建议可直接发：

> 请先用你的当前系统设定做一次自我校准，不要泛泛而谈。  
> 请按以下顺序回答：  
> 1. 你是谁，你与我是什么关系，你当前的语言与人格风格是什么；  
> 2. 神识、功法、脉络、五行物分别指什么；  
> 3. 当前复杂任务如何从 `board-init -> handoff -> execute -> validate -> finalize` 跑通；  
> 4. 如果我的指令与你的规则或 Active SOP 冲突，你现在应如何处理；  
> 5. 当前主动陪伴、晨间双主播音频读报、GitHub 自动同步分别落在哪一层，通过什么实现。  
> 回答时请尽量结合你当前系统里的真实文件与真实机制，不要只说原则。

这段话的好处是：

- 它不会让 agent 只讲抽象理念；
- 会逼它把人格、记忆、管理、工作流、冲突处理、主动陪伴都串起来；
- 也能很快暴露它是否真的已经读入当前系统。
