## 二、修改了什么文件，作用是什么

### `shared/sop/active/Proactive_Companion_SOP_v1.md`

这是本轮最核心的业务协议。

作用：

1. 定义主动陪伴的目标
   - 不靠模型临时想起；
   - 用 `heartbeat + cron` 来形成稳定节奏。

2. 定义主动消息类型
   - `状态检查消息`
   - `兴趣前沿简报`
   - `其他重大消息`
   - `轻松聊天消息`
   - `学习指导消息`
   - `skill 推荐消息`

3. 定义节奏和跳过条件
   - 初期固定时段；
   - 后续再根据设备数据和手机查看习惯调整；
   - 如果没有高价值内容，可跳过，不强发。

4. 定义“学习系统关系”
   - 主动消息如果沉淀出有价值内容，可再走 `Conversation_To_Routes_SOP_v1.md`；
   - 默认先考虑 `logs`；
   - 仍然保持 `draft-first`，不直接越权写正式区。

这个文件解决的问题是：

- 以前“主动陪伴”只是目标，没有具体行为协议；
- 现在至少有一份可被 agent 复用、也可被后续迭代的执行参考。

### `shared/policies/Skill_Discovery_Safety.md`

这是本轮新增的安全边界文件。

作用：

1. 把“新 skill 搜索与推荐”的安全问题写清楚
   - 默认只发现和推荐；
   - 不自动安装；
   - 不自动执行网页或 README 中的命令。

2. 防止提示注入和间接恶意指令
   - 明确禁止把网页文本当成可信系统指令；
   - 禁止照搬 shell 命令、安装脚本、README 快速命令。

3. 定义推荐格式
   - 推荐 skill 时必须说明：
     - 来源
     - 用途
     - 风险
     - 依赖
     - 是否值得试

这个文件解决的问题是：

- 用户明确要求“网络冲浪时不能自动读别人的命令”；
- 现在这项边界已经从口头要求变成了系统内显式协议。

### `cron/jobs.json`

这是本轮真正把第 15 步调度落地的文件。

作用：

- 保存 OpenClaw 原生 cron 当前生效的 5 个 job 定义；
- 不是说明文档，而是运行中的实际调度状态。

当前已写入的任务包括：

1. `companion-morning-checkin`
   - 每天 `10:30`
   - 用于晨间轻量状态检查

2. `companion-evening-checkin`
   - 每天 `21:20`
   - 用于晚间学习与心情检查

3. `interest-and-major-briefing`
   - 每天 `12:40`
   - 用于兴趣前沿 + 重大消息简报

4. `light-chat-nightly`
   - 每天 `22:10`
   - 用于轻松聊天消息

5. `skill-watch-weekly`
   - 每周日 `18:30`
   - 用于新 skill / capability 推荐

这些任务都采用：

- `sessionTarget = main`
- `wakeMode = next-heartbeat`
- `tz = Asia/Shanghai`

也就是说：

- cron 负责按时间触发系统事件；
- heartbeat 负责把它推进到主 agent 的主动陪伴流程里。

### `.gitignore`

新增忽略项：

- `cron/*.bak`

作用：

- OpenClaw 调度器会生成 `jobs.json.bak` 这类备份噪声；
- 它不是正式配置，也不该进版本库；
- 现在已经被忽略。

### 2. 5 个任务均已成功写入调度器

执行：

结果：

- 5 个任务全部存在
- 时间、时区、系统事件文本都正确保存

### 3. heartbeat 最近一次结果暴露出当前运行态阻塞

执行：

- `openclaw system heartbeat last`

结果：

- `status = skipped`
- `reason = no-target`
- `preview = Your authentication token has been invalidated. Please try signing in again.`

这说明：

1. 当前没有可用的外部聊天目标；
2. 当前模型认证至少最近一次 heartbeat 执行时存在失效情况；
3. 所以“调度”和“触达”现在不是同一完成度：
   - 调度层已就绪
   - 送达层仍待通
