### 本轮给你的实际操作建议

你现在先不要自己设 primary / fallback。

你只做“接 provider + 把可明确的模型白名单加进去”这一步：

#### Moonshot

先接 token：

```bash
openclaw models auth paste-token --provider moonshot
```

然后把选中的模型加入 agent 允许列表：

```bash
openclaw config set 'agents.defaults.models["moonshot/kimi-k2.5"]' '{}' --strict-json
```

#### Qwen

先启用 plugin：

```bash
openclaw plugins enable qwen-portal-auth
openclaw gateway restart
```

然后做 OAuth 登录：

```bash
openclaw models auth login --provider qwen-portal
```

再把选中的模型加入 agent 允许列表：

```bash
openclaw config set 'agents.defaults.models["qwen-portal/coder-model"]' '{}' --strict-json
```

#### DeepSeek

你这一步先只接 token：

```bash
openclaw models auth paste-token --provider deepseek
```

这次先不要自己写：

```bash
agents.defaults.models["deepseek/..."]
```

原因：

- 当前本地 catalog 没有直接列出 `deepseek` 模型
- 说明这家在你这版 OpenClaw 里至少不是“像 moonshot 一样清晰可枚举”
- 所以这一步先把 auth 接通，后面我再统一做 smoke test 和模型落位
