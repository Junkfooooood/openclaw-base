## 二、修改了什么文件，作用是什么

### `.openclaw` 仓库内

#### `extensions/obsidian-bridge/index.mjs`

这是本轮最核心的改动，作用有 4 个：

1. 新增默认业务路由表
   - 内置了 `logs / knowledge / taskboard / capabilities / reputation / strategy` 六个 route。
   - 每个 route 都绑定了：
     - `label`：给人看的业务名称；
     - `draftSubdir`：草稿应写入 `Drafts/AI` 下的哪个子目录；
     - `officialRoot`：正式区对应哪个 Obsidian 根目录。
   - 这解决了“后续 agent 必须硬编码中文路径”的问题。

2. 新增 route alias 解析
   - 现在不只支持英文 route key，也支持常见中文别名，例如：
     - `日志` -> `logs`
     - `知识库` -> `knowledge`
     - `任务看板` -> `taskboard`
     - `六维能力` -> `capabilities`
   - 这解决了你后续在对话中直接用中文业务名时，CLI / tool 仍然能正确落路由。

3. 新增 route-aware 命令和工具
   - 新增 `obsidian_bridge_routes`
     - 用来返回当前生效的业务路由表。
   - 新增 `obsidian_bridge_write_routed_draft`
     - 用 route key 直接把草稿写进对应的草稿目录，例如：
       - `logs` -> `Drafts/AI/Logs`
       - `knowledge` -> `Drafts/AI/Knowledge`
   - 新增 `obsidian_bridge_prepare_routed_patch`
     - 用 route key 直接生成正式区 patch 草稿，例如：
       - `knowledge` 会把目标路径自动挂到 `_知识库/...`
   - 这解决了“每次写草稿都要自己拼完整目标路径”的问题。

4. 修复多行文本的换行问题
   - 新增 `normalizeMultilineText`
   - 现在通过 CLI 传入的 `\\n` 会被还原成真正的换行
   - 这解决了 route smoke test 第一版 patch 中出现的“Markdown 内容全是反斜杠字面量”的问题
