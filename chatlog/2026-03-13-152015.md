## 一、本轮目标

本轮目标是补一条“换模型 / 换账号 / 新会话接管时”的强制校准 hook。

你提出的问题是：

- 如果以后切换大模型，或者 Codex 登录了不同账号，新模型是否还会天然记得之前的聊天记录？

答案是否定的，因此你要求系统本身补一层强制机制：

- 每次切换或接管时，都必须先看记忆库与人格库，不能把“模型可能记得”当作依赖。

## 二、本轮完成了什么

### 1. 找到了现成可用的真正 hook 入口

本轮我核对了 OpenClaw 当前 hook 能力，确认现有可用入口包括：

- `session-memory`
- `command-logger`
- `boot-md`
- `bootstrap-extra-files`

其中真正适合“新会话强制注入必读文件”的，是：

- `bootstrap-extra-files`

因为它会在：

- `agent:bootstrap`

事件发生时，把指定的 bootstrap 文件直接注入 agent 上下文。

### 2. 新增了 `workspace-main/BOOTSTRAP.md`

本轮新增：

- `workspace-main/BOOTSTRAP.md`

它现在明确要求：

1. 把每次新会话、切换模型、切换账号、上下文接管都视为“刚醒来”；
2. 不得假设自己天然记得旧对话；
3. 必须先读：
   - `IDENTITY.md`
   - `SOUL.md`
   - `USER.md`
   - `AGENTS.md`
   - `MEMORY.md`
   - `HEARTBEAT.md`
4. 必须先确认：
   - 我是谁

   - 冲突该如何上报并由林拍板
   - 当前 Git / 晨报 / 管理主链这些关键事实
