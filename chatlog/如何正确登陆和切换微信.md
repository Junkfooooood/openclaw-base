# 如何正确登陆和切换微信

## 一、先搞清楚这件事在做什么

当前这套微信接入，不是把你的主号“授权给机器人临时用一下”，而是：

- 让某一个微信号，正式成为 OpenClaw 在微信里的消息入口。

也就是说：

- 谁扫二维码登录，谁就是“机器人号”
- 别人以后给这个号发消息，OpenClaw 才能收到
- OpenClaw 回消息，也会从这个号发出去

因此建议：

- 优先使用副号 / 测试号
- 不要一开始就直接用主号接

## 二、如何正确登录微信号

### 第 1 步：先取当前二维码链接

在 Mac mini 终端执行：

```bash
bash /Users/linqingxuan/.openclaw/shared/runtime/wechat/latest_wechat_qr.sh
```

它会输出一个二维码页面链接。

### 第 2 步：打开二维码页面

把终端输出的链接复制到浏览器打开。

如果旧二维码失效，就重新执行上面的命令，取最新的链接。

### 第 3 步：用“准备作为机器人号的微信”去扫码

这一点最关键：

- 扫码的那个微信号，就是以后机器人在微信里的身份。

不是“你本人主号授权一下”，而是“这个号本身成为机器人号”。

### 第 4 步：在手机上确认登录

扫码后，如果微信弹出确认或授权提示，需要在手机上明确点确认。

只扫码不确认，通常不会真正完成登录。

### 第 5 步：等待同步

确认后，等待约 `5-15` 秒，让：

- WeChatPadPro
- OpenClaw gateway
- wechat channel monitor

把登录态同步完成。

### 第 6 步：用另一个微信号发起测试

不要让机器人号自己和自己说话。

应该用“另一个微信号”去给机器人号发消息。

第一条建议发：

```text
@ai 你好
```

当前默认配置里：

- 触发前缀是 `@ai`

不带这个前缀，机器人通常不会响应。

### 第 7 步：如出现配对码，就批准

当前私聊策略是：

- `dmPolicy = pairing`

所以第一次有人私聊机器人号时，系统可能会回一个配对码。

此时在 Mac mini 终端执行：

```bash
openclaw pairing approve wechat <配对码>
```

批准后，再发一次：

```text
@ai 你现在是谁
```

若正常回复，则说明微信接入成功。

## 三、如何判断是否真的登录成功

可在终端执行：

```bash
grep '\[wechat\]' ~/.openclaw/logs/gateway.log | tail -n 30
```

若看到类似内容，通常说明登录已成功或接近成功：

- `login confirmed`
- `starting monitor`
- `polling for messages`

若一直反复出现：

- `waiting for QR code scan`
- `login not confirmed — will retry`

那通常表示：

- 还没扫码
- 扫了但没确认
- 或二维码已过期

## 四、以后如何切换微信号

“换号”本质上就是：

- 让 WeChatPadPro 不再使用当前已登录微信
- 然后重新显示一个新的登录二维码
- 再让另一个微信号去扫

### 最简切换法

如果只是想从当前号切到另一个号，先执行：

```bash
docker restart openclaw-wechatpadpro
```

然后重新取二维码：

```bash
bash /Users/linqingxuan/.openclaw/shared/runtime/wechat/latest_wechat_qr.sh
```

再用新的微信号扫码。

### 更稳的切换法

如果你怀疑旧登录态残留，建议直接重新拉起整套栈：

```bash
bash /Users/linqingxuan/.openclaw/shared/runtime/wechat/start_wechat_stack.sh
```

然后再执行：

```bash
bash /Users/linqingxuan/.openclaw/shared/runtime/wechat/latest_wechat_qr.sh
```

### 切换后的验收方式

切换新号后，仍然按登录验收流程走：

1. 用另一个号给机器人号发：

```text
@ai 你好
```

2. 若要求配对，则批准：

```bash
openclaw pairing approve wechat <配对码>
```

3. 再发：

```text
@ai 你是谁
```

## 五、最常见的误区

### 误区 1：我扫的是“授权给机器人使用我的主号”

不是。

更准确地说：

- 你扫上的那个号，就是机器人号本身。

### 误区 2：登录后要从同一个号给自己发消息测试

不对。

应该使用另一个微信号去找机器人号聊天。

### 误区 3：发普通消息就会回复

不一定。

当前默认需要触发前缀：

- `@ai`

### 误区 4：二维码过期了还能继续扫

不行。

二维码过期后，应重新执行：

```bash
bash /Users/linqingxuan/.openclaw/shared/runtime/wechat/latest_wechat_qr.sh
```

取最新链接。

## 六、当前推荐做法

如果你现在只是测试，推荐这样做：

1. 先用副号扫登录
2. 用另一个号发 `@ai 你好`
3. 若出现配对码，就批准
4. 验收通过后，再决定是否长期保留这个号作为机器人号

一句话总结：

- 扫码的号 = 机器人号
- 测试消息要从另一个号发
- 第一条先发 `@ai 你好`
- 要换号就重启 `wechatpadpro` 后重新扫码
