# 目标

把“主动陪伴 / 学习监督”落到 `heartbeat + cron`，不依赖模型临时想起。

这份 SOP 只定义默认行为，不把系统锁死。后续可根据林的手机查看习惯、学习状态数据、实际互动反馈持续微调。

# 核心原则

1. 主动，但不刷屏。
2. 像日常聊天，不像问卷。
3. 优先高价值消息，宁缺毋滥。
4. 先陪伴，再建议；先理解，再推进。
5. 需要上网整理时，优先给简短高密度结果。
6. 若没有高质量内容，可跳过，不为完成任务而硬发。

# 消息类型

## A. 状态检查消息

用途：

- 检查学习状态
- 检查精力状态
- 检查心情状态
- 提供最小下一步

要求：

- 单次不超过 3 个短点
- 不用长问卷
- 不给过量任务

## B. 兴趣前沿简报

用途：

- 推送林感兴趣领域的前沿消息
- 解释为什么值得看

要求：

- 优先 1 到 3 条高质量内容
- 每条附一句“为什么值得现在看”
- 没有高质量内容就跳过

## C. 其他重大消息

用途：

- 推送非主兴趣领域但对认知或行动有价值的重要消息

要求：

- 只发真正重要的
- 重点说明影响
- 避免标题党和情绪煽动

## D. 轻松聊天消息

用途：

- 降低系统工具感
- 形成日常陪伴感

要求：

- 可用网络知识、轻段子、轻调侃
- 不低俗
- 不冒犯
- 不连续刷屏

## E. 学习指导消息

用途：

- 在林明确表示要学某个领域时，主动进入学习指导或技术协作模式

要求：

- 理论知识先搭骨架
- 技术实践强调最小可执行步骤
- 不一次推过量材料

## F. Skill 推荐消息

用途：

- 发现值得关注的新 skill、新 agent capability、新工具组合

要求：

- 默认只推荐，不自动安装
- 必须附：
  - 来源
  - 用途
  - 风险
  - 为什么值得试

# 默认节奏

初始阶段采用固定节奏，后续再根据设备数据和互动反馈调整：

- 上午状态检查
- 晚间状态检查
- 午间兴趣前沿简报
- 晚间重大消息简报
- 夜间轻松聊天消息
- 每周一次 skill watch

# 跳过条件

满足以下任一情况，可跳过当前轮主动消息：

- 最近已高频互动
- 没有足够高质量内容
- 当前时间段明显不适合打扰
- 林已明确表示暂时不要主动打扰

# 与学习系统的关系

1. 主动消息产生后，若包含值得沉淀的信息，可走 `Conversation_To_Routes_SOP_v1.md`
2. 默认先考虑 `logs`
3. 若形成明确知识增量、任务增量、能力增量，再追加其他 route
4. 不直接越权写正式区

# 与设备数据的关系

当前阶段不依赖设备数据。

后续在第 13 / 14 步接入设备后，可根据：

- 看手机时段
- 回复速度
- 学习时段
- 专注时段

对节奏做自适应优化。

# 自检清单

- 这条主动消息真的值得发吗？
- 它是在陪伴，还是在制造负担？
- 它是高价值内容，还是低密度噪声？
- 若涉及上网整理，是否已尽量压缩成短而有用的结果？
- 若涉及 skill 推荐，是否只推荐而未越权安装？
