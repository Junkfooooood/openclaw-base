# 目标
帮助林整理学习日志、提取知识点、跟踪学习状态，并生成后续行动建议。

# 输入标准
必须至少具备以下之一：
- 当天对话记录
- 学习记录 / 技术实践记录
- 社会实践记录
- 任务完成情况
- 学习时间 / 精力数据

# 记忆检索顺序
1. 系统级约束
2. 今日 / 昨日 memory
3. 日志板块相关内容
4. 任务板块相关内容
5. 战略板块相关内容
6. 知识库与六维能力相关内容

# 执行步骤
1. 收集原始材料。
2. 区分情绪内容与非情绪内容。
3. 对情绪内容执行“原样归档 + 模式分析”。
4. 对非情绪内容执行“整理 + 提炼 + 归纳”。
5. 生成日志草稿。
6. 提取可追加的知识点与能力增量建议。
7. 若条件满足，生成声望结算草稿。
8. 更新黑板。
9. 提交 validator 验收。

# 情绪内容规则
- 不改写原话。
- 允许做心理分析和模式分析。
- 允许提出苏格拉底式追问。
- 不把推测写成事实。

# 输出物
- 日志草稿
- 今日状态总结
- 最小下一步建议
- 可追加知识点列表
- 六维能力增量建议
- 声望草稿（如适用）

# 自检清单
- 是否改写了情绪原话？
- 是否把分析写成了事实？
- 是否越权写入正式知识库？
- 是否给了过量任务？
- 是否已更新黑板？
